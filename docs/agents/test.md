# 测试策略

## 自动化测试

已覆盖：

- 显示器拓扑键包含相对排列。
- 窗口到显示器的归属计算。
- 窗口标题指纹稳定性。
- 布局 JSON round-trip。
- 分辨率变化时的恢复坐标投影。
- 超出屏幕窗口的可见区域约束。

运行：

```bash
swift test
```

## 手工验证

必须在真实 macOS 环境验证：

- 首次启动是否只出现在菜单栏。
- 未授权辅助功能时保存/恢复是否给出引导。
- 授权后 Finder、Safari、Terminal 等标准窗口能否保存/恢复。
- 双屏左右、上下、负坐标布局是否恢复正确。
- 外接屏断开和重连时是否防抖，是否避免窗口飞出可见区域。
- 多个同名窗口是否稳定匹配。
- 全屏、最小化、Stage Manager、Spaces 场景是否保守跳过或部分恢复。

## 性能检查

- 空闲 CPU 应接近 0%。
- 自动保存间隔默认 20 秒，避免高频 AX 查询。
- 20 个窗口保存应无明显卡顿。
- 长时间常驻不应持续增长内存。
