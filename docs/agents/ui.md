# UI 设计

## 菜单栏

首版只提供菜单栏入口，符合轻量常驻定位。

菜单结构：

```text
Winstore
显示器：N 台已连接
权限正常 / 需要辅助功能权限

保存当前布局
恢复当前显示器布局
暂停/开启自动恢复
暂停/开启自动保存

显示器详情...
打开辅助功能设置...
退出
```

## 视觉原则

- 使用 macOS 系统菜单和 SF Symbols。
- 正常状态使用 `rectangle.on.rectangle`。
- 权限缺失使用 `exclamationmark.triangle`。
- 不做网页式主界面，不做营销页。

## 状态反馈

- 保存成功：展示保存窗口数量。
- 恢复成功或部分成功：展示恢复、跳过、失败数量。
- 权限缺失：提示需要辅助功能权限。
- 显示器变化：提示等待稳定，再自动恢复或保存。

## 后续设置窗口

后续可增加设置窗口，采用左侧导航：

- 布局
- 显示器
- 权限
- 偏好设置
- 关于

MVP 不实现拖拽式布局编辑器，避免首版复杂度扩散。
